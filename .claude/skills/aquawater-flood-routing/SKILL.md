# 水库调洪演算 Skill

## 概述

本 Skill 用于执行水库调洪演算——模拟水库在洪水来临时，水位和泄流量的动态变化过程。

**核心物理方程**：水量平衡原理

```
dV/dt = I(t) - Q(Z)
```

其中 `I(t)` 为入库流量 (m³/s)，`Q(Z)` 为水位 Z 对应的泄流能力 (m³/s)，V 为库容 (万 m³)。水位 Z 与库容 V 通过库容曲线关联。

## 输入要求

### 三组数据曲线（二列文本文件，Tab/逗号/空格分隔）

| 曲线 | 第一列 | 第二列 | 示例 |
|------|--------|--------|------|
| **库容曲线** (ZV) | 水位 (m) | 库容 (万 m³) | `scripts/sample_data/库容曲线.txt` |
| **泄流曲线** (ZQ) | 水位 (m) | 最大泄流 (m³/s) | `scripts/sample_data/泄流.txt` |
| **洪水过程线** (Flood) | 时间 (h) | 流量 (m³/s) | `scripts/sample_data/P=5%洪水过程线.txt` |

### 四个边界条件参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| 起调水位 (m) | 洪水开始时的水库水位 | 1024 |
| 防洪限制水位 (m) | 低于此水位时的特殊处理（可选，默认 0） | 0 |
| 计算时段 (h) | 时间步长，越小越精确 | 0.1 |
| 演算方法 | 0=积分法(专利公式), 1=龙格库塔法, 2=迭代法(二分) | 0 |

## 工作流程

### 步骤 1：获取用户数据

用户可能以以下方式提供数据：
- **文件路径**：用户指定 `.txt` 文件路径
- **粘贴数据**：用户直接在对话中粘贴数据
- **口头描述**：用户描述水库特征，你根据规范估算参数

**读取数据文件**：使用 Read 工具读取用户指定的文件，根据数据格式（二列数字）自动解析。

**如果用户没有提供数据**：主动询问三个曲线数据和四个参数，一次问一项，不要一次全问。

### 步骤 2：运行计算

使用 skill 自带的 `scripts/calculate.py` CLI 脚本：

```bash
python <skill-dir>/scripts/calculate.py \
  --zv <库容曲线文件> \
  --zq <泄流曲线文件> \
  --flood <洪水过程线文件> \
  --z-start <起调水位> \
  [--z0 <防洪限制水位>] \
  [--dt <计算时段>] \
  [--method <0|1|2>] \
  [--chart <输出图表HTML路径>]
```

脚本输出 JSON 格式的计算结果到 stdout，包含：
- `summary`: 特征值（起调水位、起调库容、最高水位、最大库容、最大泄流、滞洪库容）
- `results`: 逐时段成果表（时间、平均入库、平均出库、时段初水位、时段末水位、时段初库容、时段末库容）
- `chart_data`: 图表数据（时间轴、入库流量序列、出库流量序列、水位序列）

**另外**：如果用户要求生成可视化图表，加 `--chart result.html` 参数生成独立的 ECharts 交互式 HTML 图表文件。

### 步骤 3：展示结果

计算结果以以下格式呈现给用户：

#### 3a. 特征值摘要表

```
📊 调洪演算成果 — 特征值
┌──────────────────────────┬─────────────┐
│ 起调水位                 │ XXXX.XX m   │
│ 起调库容                 │ XXXX.XX 万m³│
│ 调洪最高水位 ⬆           │ XXXX.XX m   │
│ 调洪最大库容             │ XXXX.XX 万m³│
│ 调洪最大下泄流量         │ XXXX.XX m³/s│
│ 滞洪库容                 │ XXXX.XX 万m³│
└──────────────────────────┴─────────────┘
```

#### 3b. 逐时段成果表（前 10 行 + "...共 N 行"）

如果用户需要完整数据，提供 Excel 导出（使用 openpyxl）。

#### 3c. 图表（可选）

如果生成了图表 HTML，告知用户文件路径，用户可在浏览器中打开查看交互式图表（入库/出库流量 + 水位三条曲线，双Y轴）。

## 三种演算方法说明

| 方法 | 编号 | 原理 | 适用场景 |
|------|------|------|----------|
| 积分法（专利公式） | 0 | 将 S-Q 关系近似为线性求解析解 | **推荐默认**，速度快精度高 |
| 龙格库塔法（四阶） | 1 | 经典 RK4 数值求解 ODE | 理论严密，精度高 |
| 迭代法（二分法） | 2 | 猜试水位直到收敛 | 最直观，但可能不收敛 |

## 硬约束

- **数据单调性检查**：库容曲线和泄流曲线必须单调递增。如果用户数据不是单调的，警告用户并指出问题点。
- **水位范围**：起调水位必须在库容曲线的水位范围内，否则外推可能不准，需提醒用户。
- **时间步长**：建议 dt ≤ 0.5h，过大的步长会导致精度下降。
- **防洪限制水位**：如果设为 0（默认），表示不启用防洪限制逻辑。

## 常见问题处理

- **计算结果不合理**（如水位突变）：检查数据曲线是否单调、步长是否过大、单位是否正确
- **迭代法不收敛**：返回 0 值出库流量，提示用户切换到积分法或龙格库塔法
- **数据文件编码问题**：自动处理 UTF-8 BOM 和 GBK 编码
